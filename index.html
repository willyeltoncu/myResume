Car Driving Agent, Project Team Member | CU Boulder | Spring 2022
Machine Learning

Created an agent which could successfully navigate a randomly generated track in a simple racing game from the OpenAI Gym package. We created an autonomous car in a sense, within the simple CarRacing-v1 environment. After a few weeks of trial and error the project team ended up using the algorithm known as NeuroEvolution of Augmented Topology, more simply the NEAT algorithm. Processed image data is used as the input to a recurrent neural net and the optimal action is the output of the NN. We created a score heuristic so that the agent had a reward it was attempting to maximize. Creating the score heuristic was by far the most time consuming part of the project, mostly trial and error. After a finding reasonably good scoring criteria our Agent was able to learn and drive pretty quickly. Through our trial and error with scoring the training agent we were able to find two very good models. Model #1 is the safety focused model, this model will sacrifice its driving speed to increase the safety and reliability of the car and its passengers. Model #1 has never crashed within our tests and it would be the model we used if this agent was deployed within a real world scenario. Model #2 is the speed focused model. Model #2 is able to achieve better lap times when it functions correctly, it is a good model and the best performing racing agent we were able to create. Speed and safety are very obvious trade offs when you begin to tune the scoring function. Simply put, Model #1 is a slow conservative driver who is exclusively focused on traveling safely whereas Model #2 is willing to take risks in order to get the fastest lap time possible.
